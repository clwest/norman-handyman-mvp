# Norman Handyman MVP

Two-sided handyman business app for Norman, Oklahoma.

## Architecture

| Layer | Stack | Dir |
|-------|-------|-----|
| Backend | Django 5.1 + DRF + PostgreSQL | `/backend` |
| Customer Web | Next.js 15 (App Router, TypeScript, Tailwind) | `/web` |
| Operator Mobile | Expo (React Native, TypeScript, expo-router) | `/mobile` |

## Quick Start

```bash
# 1. Database
docker compose up -d

# 2. Backend (terminal 1)
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # Edit Stripe keys
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver

# 3. Web (terminal 2)
cd web && npm install && npm run dev

# 4. Mobile (terminal 3)
cd mobile && npm install && npx expo start

# 5. Stripe webhooks (terminal 4)
stripe listen --forward-to localhost:8000/api/webhooks/stripe/
```

## Env Vars

### backend/.env
- `DJANGO_SECRET_KEY` — change in production
- `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT` — Postgres connection
- `STRIPE_SECRET_KEY` — `sk_test_...`
- `STRIPE_WEBHOOK_SECRET` — `whsec_...` (from `stripe listen` output)
- `EMAIL_BACKEND` — `django.core.mail.backends.console.EmailBackend` (dev)
- `CORS_ALLOWED_ORIGINS` — `http://localhost:3000,http://localhost:8081`

### web/.env.local
- `NEXT_PUBLIC_API_BASE_URL` — `http://localhost:8000/api`

### mobile/.env
- `EXPO_PUBLIC_API_BASE_URL` — `http://localhost:8000/api` (emulator) or `http://<LAN-IP>:8000/api` (device)

## Data Model

`Customer` → `BookingRequest` → `Job` → `Estimate` / `Invoice` → Stripe payment

Statuses:
- BookingRequest: NEW → CONTACTED → SCHEDULED → CLOSED
- Job: SCHEDULED → IN_PROGRESS → COMPLETED → CANCELED
- Estimate: DRAFT → SENT → APPROVED → REJECTED
- Invoice: DRAFT → SENT → PAID → OVERDUE → VOID

## API

Auth: DRF TokenAuthentication. Get token: `POST /api-token-auth/` with `{username, password}`.

### Public (no auth)
- `POST /api/booking-requests/public/` — submit booking
- `GET /api/invoices/public/{id}/` — view invoice
- `POST /api/invoices/public/{id}/pay/` — create Stripe checkout

### Operator (auth required)
- `/api/customers/`, `/api/booking-requests/`, `/api/jobs/`, `/api/estimates/`, `/api/invoices/`, `/api/expenses/`, `/api/supplies/`
- Custom actions: `convert_to_job`, `start`, `complete`, `send_estimate`, `send_invoice`, `mark_paid`, `create_checkout`

### Webhooks
- `POST /api/webhooks/stripe/` — `checkout.session.completed` → marks invoice PAID

## Golden Path Demo Script

1. Start all services (backend, web, mobile, stripe listen)
2. **Customer submits booking** → web `/book` form → confirmation email logged
3. **Operator sees request** → mobile Today tab → tap booking → "Convert to Job"
4. **Operator creates invoice** → Job detail → "Create Invoice" → add line items → "Create & Share Link"
5. **Customer pays** → web `/pay/{id}` → "Pay Now" → Stripe test card `4242 4242 4242 4242`
6. **Webhook fires** → invoice flips to PAID → confirmation email logged
7. **Operator sees PAID** → mobile Invoices tab shows green PAID badge

## Known Pitfalls

- **Android device**: use LAN IP not `localhost` for API URL
- **Stripe test card**: `4242 4242 4242 4242`, any future date, any CVC
- **Emails in dev**: console backend logs to terminal, not actually sent
- **CORS**: if web can't reach backend, check `CORS_ALLOWED_ORIGINS`
- **Token auth**: mobile stores token in AsyncStorage; clear app data to reset
